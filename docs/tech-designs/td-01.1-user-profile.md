## 1. 数据库变更

### 1.1 新增字段

```sql
ALTER TABLE t_user 
ADD COLUMN gender TINYINT DEFAULT 0 COMMENT '性别 0-未知 1-男 2-女',
ADD COLUMN country VARCHAR(50) DEFAULT '' COMMENT '国家',
ADD COLUMN province VARCHAR(50) DEFAULT '' COMMENT '省份',
ADD COLUMN city VARCHAR(50) DEFAULT '' COMMENT '城市',
ADD COLUMN profile_completed TINYINT DEFAULT 0 COMMENT '资料是否完善 0-否 1-是';
```

### 1.2 更新 init.sql

在 `server/src/main/resources/db/init.sql` 的 t_user 建表语句中加上这些字段。

### 2.2 核心流程

#### 更新资料流程

```
UserController.updateProfile(UpdateProfileRequest)
    │
    ▼
从 JWT Token 中获取 userId
    │
    ▼
UserServiceImpl.updateProfile(userId, request)
    │
    ├── 1. 根据 userId 查询用户
    ├── 2. 更新 nickname, avatarUrl, gender, country, province, city
    ├── 3. 设置 profile_completed = 1
    └── 4. 保存到数据库
    │
    ▼
返回更新后的用户信息
```

#### 获取用户信息流程

```
UserController.getUserInfo()
    │
    ▼
从 JWT Token 中获取 userId
    │
    ▼
UserServiceImpl.getUserInfo(userId)
    │
    ├── 根据 userId 查询用户
    └── 转换为 UserInfoResponse
    │
    ▼
返回用户完整信息
```

### 3.2 wx.getUserProfile 调用

```javascript
getUserProfile() {
  wx.getUserProfile({
    desc: '用于完善个人资料',
    success: (res) => {
      const { nickName, avatarUrl, gender, country, province, city } = res.userInfo;
      
      // 调用后端接口保存
      request('/user/profile', 'POST', {
        nickname: nickName,
        avatarUrl: avatarUrl,
        gender: gender,
        country: country,
        province: province,
        city: city
      }).then(() => {
        wx.showToast({ title: '资料更新成功', icon: 'success' });
        // 刷新页面
        this.loadUserInfo();
      });
    },
    fail: (err) => {
      wx.showToast({ title: '授权失败', icon: 'none' });
    }
  });
}
```

## 4. 注意事项

### 4.1 wx.getUserProfile 限制

- 2021年4月13日后，`wx.getUserInfo` 不再返回真实信息，必须用 `wx.getUserProfile`
- 每次调用都会弹窗，无法静默获取
- 必须由用户真实点击触发，不能在 onLoad 等生命周期自动调用

### 4.2 头像昵称敏感内容检测

微信返回的昵称和头像可能包含敏感内容，建议：
- 后端保存前做基本校验（长度、格式）
- 如果要做内容审核，可以接入微信内容安全接口

### 4.3 用户拒绝授权的处理

- 拒绝后不影响其他功能使用
- 个人中心显示默认头像和"微信用户"
- 保留"完善资料"按钮，用户可以随时重新授权
